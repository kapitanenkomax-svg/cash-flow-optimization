# Оптимизация денежного потока через управление дебиторской задолженностью

Этот проект представляет собой **практический финансовый инструмент** для снижения кассовых разрывов за счёт анализа платёжной дисциплины клиентов, автоматического выявления рисков и моделирования эффекта от управленческих решений.
Проект основан на реальном кейсе, где компания ежеквартально теряла до **15 млн ₽** из-за несвоевременной оплаты от клиентов и вынуждена была использовать дорогие краткосрочные кредиты.

---

## Проблема

- Нестабильный денежный поток → кассовые разрывы до **15 млн ₽/квартал**  
- Зависимость от овердрафта и краткосрочного финансирования  
- Просрочки по оплате от «проблемных» клиентов (среднее время оплаты > 60 дней)  
- Отсутствие системного подхода к работе с дебиторкой

---

## Решение

Скрипт `cash_flow_optimization.py` выполняет **4 ключевые функции**:

### 1. **Анализ клиентов**
- Использует данные по каждому клиенту:  
  - среднее время оплаты (`avg_days_to_pay`)  
  - количество своевременных платежей за год  
  - годовая выручка
- Оценивает платёжную дисциплину по двум критериям:
  - просрочка **более 50 дней**
  - менее **8 из 12** платежей вовремя

### 2. **Автоматическое ранжирование и выявление рисков**
- Сортирует клиентов по уровню риска (от самых медленных к надёжным)
- Выделяет **топ 20% самых проблемных клиентов**

### 3. **Генерация обоснованного отчёта**
- Сохраняет список проблемных клиентов в файл **`problematic_clients.csv`**
- Для каждого клиента указывает:
  - конкретные причины отнесения к рисковым
  - чёткую рекомендацию: **«Ввести 100% предоплату»**

> Пример обоснования:  
> _«ср. просрочка 90 дн.; только 3 из 12 платежей вовремя»_

### 4. **Моделирование денежного потока**
- Сравнивает кассовые разрывы **до и после** внедрения мер
- Оценивает:
  - снижение среднего разрыва (в рублях)
  - годовую экономию на процентах по овердрафту
- Сохраняет визуализацию в **`cash_flow_optimized.png`**
